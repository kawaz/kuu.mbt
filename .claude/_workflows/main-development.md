# main ワークスペース運用

## 概要

main ワークスペースはプロジェクトの主要な開発作業を行う場所。
他ワークスペース（review 等）からのドキュメント反映が割り込むことがある。

## 定期チェック（Cron）

### 10分毎: 他ワークスペースからの変更確認

```bash
jj log -r '(@|review@)----..@' -s --no-pager
```

- 新しいコミット（特に docs/ の変更）があれば簡潔に報告
- 変化なしなら無言（出力なし）

### 30分毎: コミット確認

作業中のコミット状態を確認し、必要に応じて整理を提案:

```bash
jj log -r '::@ & mutable()' -s --no-pager
```

- describe 未設定のコミットがあれば指摘
- 不要な空コミットがあれば abandon を提案
- コミットが大きすぎる場合は split を提案

### 1時間毎: DR 記載漏れ確認

設計判断を伴う変更が DR に記録されているかチェック:

- 新しいコンビネータ/API/アーキテクチャ変更に対応する DR があるか
- 既存 DR の内容が実装変更で古くなっていないか
- 変化なしならスキップ（出力なし）

### 3時間毎: DESIGN / DR / 実装の整合性確認

三者間の乖離を横断的にチェック:

- `docs/DESIGN.md` の記載と実装コードが一致しているか
- DR に記載された設計判断が DESIGN.md と実装の両方に反映されているか
- 新しいコンビネータ/機能が DESIGN.md に未反映でないか
- 乖離を発見したら報告し、修正コミットを提案
- 変化なしならスキップ（出力なし）

## 他ワークスペースからの変更取り込み

review@ 等から docs の変更がマージされた場合:

```bash
# 自動的に @ に含まれるので特別な操作は不要
# ただし docs/ の変更内容を把握しておくと開発に活かせる
```

## 基本動作

- コード変更、テスト追加、機能実装が主な作業
- example 作成は別ワークスペースで並列実行
- ドキュメント反映は review ワークスペースに任せる
