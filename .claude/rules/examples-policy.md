# examples/ の取り扱い

## 書き直し禁止

既存の example を新しい API に合わせて書き直してはならない。

理由: example はその時点の kuu API の書き味・課題・制限事項を記録したアーカイブ。
振り返ることで API の進化を確認できる。

## ディレクトリ構成

```
examples/
  {date}-{name}/     # 未評価 or リファレンス品質のもの
  archives/
    {date}-{name}/   # 評価済みのもの
```

- **トップ**: 未評価、またはサイト公開後もリファレンスとして利用可能なハイクオリティなもの
- **archives/**: 評価が完了し、アーカイブとして保存するもの

## 新規 example の作成

新しい API や機能の検証には、既存 example の修正ではなく新規 example を作成する。
