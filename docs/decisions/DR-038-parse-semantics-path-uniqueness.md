# DR-038: パース意味論の確定 — 「完全経路の一意性」を契約に、最長一致は廃す

> 由来: kawaz × Claude のリモートセッション議論 (2026-06)。DR-021/DR-037 を「実装契約」として確定させたもの。

## 決定

パーサの成功条件を **「入力を全消費する完全な解決経路がちょうど1本」** と定義する。

| 完全経路の数 | 結末 |
|---|---|
| 0本 | パース失敗 (保持された Error を表示) |
| 1本 | 成功 (その経路で確定) |
| 2本以上 | ambiguous エラー (パース失敗の一種) |

これは DR-021「曖昧さなく一意に解ければ成功」/ DR-037「解けた枝の数 0/1/2+」の言い換えだが、**判定粒度を「全体パス」と明示**する点が要点。

### 「最長一致」はプリミティブな規則として持たない

ボトムアップ kuu の README が掲げる「投機実行＋最長一致」は、**曖昧解消の規則ではなく創発**として整理する:

- 末尾を拾う受け皿 (positional 等) が無ければ、短く食う経路は余りが出て**未完結＝脱落**し、結果的に長い経路だけが完結する。だから「長い方が残る」ように見えていただけ。
- 受け皿があれば短い経路も完結し、**正しく ambiguous** になる。

→ エンジンが持つべき規則は **「完全経路がちょうど1本 else 失敗/曖昧」一本**。最長一致は独立規則として不要 (消す)。

### 決定的な例: `-n1.0f`

`n` の値パーサが `"1.0f"` と `"1.0"` の両方を受理する場合:

- 経路A: `n="1.0f"`、`-f` 不発 (f=false)
- 経路B: `n="1.0"`、`-f` 発火 (f=true)

両方とも全消費で完結 → **2経路 → ambiguous**。greedy に最長 (`1.0f`) を取って確定するのは**契約違反**(別経路を黙って潰している)。

帰結として **曖昧かどうかは値パーサの寛容さに依存** = 本質的に実行時判定。DR-021「一意性検査は実行時、静的は warn のみ」がこれで完全に裏付けられる。

## エンジンへの含意 (実装契約)

- 既存 `parse_raw` は各位置で `best_consumed` を選んで **commit して前進 (greedy・バックトラックなし)**。`install_short_combine_node` も**値トライアル長い方先勝ちコミット**。どちらも**第2経路を検出できない**ため、この契約を満たさない。
- 合流エンジンは **bounded path-search**: 完全解を探索し、**2本目の完全解を見つけた時点で打ち切って ambiguous**。2本目で止められるので有界 (ただし病的文法向けに探索上限/打ち切りの設計は要)。
- 再利用可能: `TryResult{Accept/Reject/Error}` (DR-037 の Reject/Error は既存実装に存在)、`make_or_node` の Reject/Error 構造。
- 置換対象: `parse_raw` の greedy-commit ループ、`install_short_combine_node` の先勝ちコミット。

## 受け入れる帰結

1. **greedy より多くの文法が ambiguous になる**。可変アリティ option ＋ 後続 positional は、アリティが一意化を強制しない限り本質的に曖昧 (これは正しい曖昧検出。`mv a b c dst` は dst=ちょうど1個でアリティが固定され一意)。
2. **曖昧性は値パーサ依存 → 実行時判定**。静的バリデータは構造的リスクの warn まで (DR-021)。
